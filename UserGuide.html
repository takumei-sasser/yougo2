用語追加マニュアル (Glossary App)

このアプリケーションに新しい用語を追加するための手順を解説します。

STEP 1: データの登録 (app.js)

app.js を開き、const terms = [...] 配列の中に新しいオブジェクトを追加します。

{ 
    id: 'zos-new-term',      // ID (一意であること。HTMLファイル名と合わせるのがおすすめ)
    title: '新しい用語名',     // 画面に表示されるタイトル
    file: 'zos-new-term.html',// content/フォルダ内に作成するファイル名
    category: 'Core'         // カテゴリー (Core, Storage, Subsystems 等)
},

STEP 2: 解説ファイルの作成 (content/)

content/ フォルダ直下に、STEP 1で指定したファイル名（例: zos-new-term.html）で新規作成します。以下のテンプレートをコピーして使用してください。

<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <link rel="stylesheet" href="../style.css">
    <script src="../mermaid.min.js"></script>
    <script>
        window.addEventListener('load', () => {{
            mermaid.initialize({{ startOnLoad: true, theme: 'neutral', securityLevel: 'loose' }});
            mermaid.init();
        }});
    </script>
</head>
<body class="is-iframe">
    <h1>用語タイトル</h1>
    <p>ここに詳細な解説文を記述します。</p>
    
    <!-- 図解を入れたい場合 (任意) -->
    <div class="mermaid">
    graph LR
        A[要素A] --> B[要素B]
    </div>
</body>
</html>

💡 重要なポイント

  • パスの指定: content/ 内部のファイルから見て、CSSやJSは親フォルダにあるため、必ず ../style.css のように ../ を付けてください。
  • Mermaidの描画: ローカル環境で図解が表示されない場合は、npx serve 等でローカルサーバーを起動して閲覧してください。
  • ヒーローセクション: 現在、上部の絵文字背景（term-hero）は廃止されたため、HTML内に記述する必要はありません。

✨ これで新しい用語がサイドバーのリストに自動的に反映されます！
