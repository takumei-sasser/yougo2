Dataset

z/OSにおけるファイル。複数の形式（PS, PDS, VSAMなど）があります。

スペシャリストによる技術解説

z/OSにおけるデータセットは、単なるバイト列ではなく、レコード単位で管理される構造化されたデータの一群です。その物理的な属性は VTOC (Volume Table of Contents) 内の DSCB (Data Set Control Block) に記録されます。

  • レコード形式 (RECFM): 固定長（F/FB）、可変長（V/VB）、および不定形式（U：主にロードモジュール用）があります。FB（Fixed Blocked）では、複数のレコードを一つの物理的なブロックにまとめることで、I/O効率を向上させます。
  • ブロックサイズの最適化: 現代的なシステムでは RECFM=FB, BLKSIZE=0 と指定することで、DASDデバイスのトラック容量に合わせてシステムが自動的に最適なブロックサイズ（System Determined Block Size）を決定します。
  • エクステント (Extent): データの増大に伴い、DASD上の非連続な領域が割り当てられる単位です。一つのデータセットは通常最大16個のエクステント（SMS管理下ではそれ以上）を持つことができ、これを超えると B37 等のアベンドが発生します。
