VTAM (Virtual Telecommunications Access Method)

メインフレームにおける SNA（Systems Network Architecture）ネットワークの中核を担う通信アクセス方式です。Net & Secカテゴリーに分類されます。

役割と特徴

VTAM は、アプリケーション（例: CICS, IMS, TSO）とネットワーク上の端末や他のシステムを接続する司令塔です。

  • SNA の実装: ネットワークリソース（論理装置 LU、物理装置 PU）の定義、セッションの確立、ルーティング制御などを司ります。
  • APPN (Advanced Peer-to-Peer Networking): 従来の階層型ネットワークから、より動的でルーティングの柔軟性が高い通信をサポートします。
  • TCP/IP との統合 (Enterprise Extender): 現在のメインフレーム・ネットワークは IP ベースが主流ですが、従来の SNA アプリケーションを改修せずに IP ネットワーク上で通信させるためのカプセル化（UDP/IP 上に SNA パケットを乗せる）機能も担っています。

Communication Server との位置づけ

現在、z/OS において VTAM と TCP/IP は「z/OS Communications Server」という統合的なネットワーキング基盤の双璧をなすコンポーネントとして提供されています。システム起動時には TCP/IP の前に必ず VTAM が起動されている必要があります。
