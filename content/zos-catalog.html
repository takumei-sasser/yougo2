Catalog

データセットの名前と、それが格納されているボリューム（場所）を紐付けて管理するデータベース。

スペシャリストによる技術解説

z/OSのカタログシステムは ICF (Integrated Catalog Facility) と呼ばれ、データセットの物理的な場所を意識せずにアクセス（Device Independence）することを可能にします。

  • Master Catalog vs User Catalog: システムには一つの Master Catalog が存在し、重要なシステムデータセットを管理します。一般のデータセットは User Catalog に登録され、Master Catalog 内に定義された ALIAS（名前の接頭辞、HLQ）によって、どのUser Catalogを検索するかが決定されます。
  • ICFの構造:
    • BCS (Basic Catalog Structure): カタログ名やボリューム情報を保持するVSAM KSDS。
    • VVDS (VSAM Volume Data Set): 各ボリュームに存在し、そのボリューム内のVSAMデータセットの詳細な属性（CIサイズ等）や、非VSAMデータセットのカタログ連携情報を保持するデータセット。
  • カタログ検索の最適化: カタログは CAS (Catalog Address Space) によって管理され、メモリ上に情報をキャッシュすることで、ジョブ開始時のステップ割り当て（Allocation）を高速化します。
