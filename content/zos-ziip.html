zIIP (IBM Z Integrated Information Processor)

zIIPは、特定のワークロード（Java、XML処理、Db2のネットワーク処理、システム管理の一部など）をメインのプロセッサからオフロードして実行するための専用プロセッサです。zIIPで実行されるワークロードについてはソフトウェアライセンス料が軽減されるため、コスト効率の向上に大きく寄与します。

主な特徴

  • コスト削減: zIIPで処理される時間は、MSU（ソフトウェア課金の指標）の計算に含まれません。
  • パフォーマンス向上: 本体のプロセッサ（GP）の負荷を軽減し、システム全体の効率を高めます。
  • 自動化: OS（z/OS）が自動的に適切な処理をzIIPに振り分けます。

スペシャリストによる技術解説

zIIPは、SRB (Service Request Block) モードで動作する特定のワークロードをオフロードするために設計されています。一般的なTSOユーザーのタスク（TCB）が直接zIIPで動くことはありません。

  • Enclave (アンクレイブ): Db2の分散クエリ計算などは、Enclaveと呼ばれるWLMの管理単位として扱われ、その一部（通常は最大100%に近い割合）がzIIPにスイッチして実行されます。
  • IIPHONORPRIORITY: SYS1.PARMLIB(LOADxx) や WLMの設定で、GPに余裕がある場合にzIIPワークロードをGPで助けるかどうかを制御します。これをNOに設定すると、zIIPワークロードはzIIPプロセッサが空くまで待機し、GPのリソースを消費しません。
  • zIIP-Exploitation: Java (JZOS), z/OS XML System Services, Global Mirror (XRC) のシステムタスクなどが主要な利用者です。最近ではContainer Extension (zCX) や、Spark/AI Toolkit などのモダンなワークロードもzIIPを最大限に活用します。
  • SMF記録: SMFタイプ30レコードなどを解析することで、実際にどの程度の「zIIP on CP」（zIIPで動くべきだったがCPで動いた時間）が発生しているかを把握し、zIIP増設のキャパシティプランニングに役立てます。
