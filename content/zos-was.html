WebSphere Application Server

WebSphere Application Serverは、z/OS（メインフレーム）を支える重要な技術要素の一つです。Middlewareカテゴリーに分類されます。

スペシャリストによる技術解説

WebSphere Application Server (WAS) for z/OS は、分散系サーバーでも普及しているJava EEアプリケーションサーバーですが、z/OS版はOSのコア機能と深く統合された特殊なアーキテクチャを持ちます。

  • Controller/Servant アーキテクチャ: 単一の巨大なJVMではなく、リクエストをキューイングし管理する Controller (制御領域) と、実際にJavaアプリを実行する複数の Servant (付帯領域) に分かれて動作します。異常終了したServantはWLMによって瞬時に再起動され、システム全体がダウンすることはありません。
  • WLM (Workload Manager) との密連携: HTTPリクエストの到着時、WLMのポリシーと目標（レスポンスタイム等）に基づいて、動的にServant領域の数を増減（スケールアウト/イン）させます。
  • zIIPエコノミー: JVM上で動作するJavaコードは、特権を持たない限り zIIP (System z Integrated Information Processor) にほぼ100%オフロード可能であり、ソフトウェアライセンス費用の抑制に大きく貢献します。
  • セキュリティ統合: SAFインタフェースを通じ、JavaのJ2EEセキュリティロールを直接 RACF と統合して認証・認可を行えます。
